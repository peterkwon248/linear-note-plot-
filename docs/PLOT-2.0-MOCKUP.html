React

UI를 구축하기 위한 JavaScript 라이브러리. Meta(전 Facebook)에서 개발하며 컴포넌트 기반 + 선언적 패러다임을 핵심으로 한다.

Aliases: ReactJS React.js

1. 개요

React는 2013년 Meta(당시 Facebook)에서 오픈소스로 공개한 UI 라이브러리이다. 2025년 기준 가장 널리 사용되는 프론트엔드 라이브러리 중 하나이며, Next.js, Remix 등 대규모 프레임워크의 기반으로 활용된다.

컴포넌트 기반 아키텍처와 단방향 데이터 흐름을 통해 복잡한 UI를 작은 단위로 분해하여 관리할 수 있도록 설계되었다.

2. 특징

2.1. JSX

JSX는 JavaScript의 확장 문법으로, HTML과 유사한 구문으로 UI를 표현할 수 있게 해준다. 빌드 단계에서 React.createElement 호출로 변환된다.

2.2. 가상 DOM

React는 메모리 상에 실제 DOM의 가벼운 사본인 가상 DOM을 유지한다. 상태 변경 시 가상 DOM에서 변경 사항을 비교(diffing)한 후 최소한의 실제 DOM 조작만 수행하여 성능을 최적화한다.

2.3. Hooks

React 16.8에서 도입된 Hooks는 함수형 컴포넌트에서 상태 관리와 생명주기 기능을 사용할 수 있게 해준다. useState, useEffect 등이 대표적이다.

Plot 2.0 리디자인 전략

Plot은 지금 안정적이지만, **정보 밀도와 시각적 균형**이 부족하다. 사용자가 노트를 빨리 훑고, 컨텍스트를 잡고, 깊이 들어갈 수 있어야 한다. Linear의 속도, Notion의 유연함, Reflect의 미니멀함 — 이 셋의 균형점이 목표다.

핵심 변화 3가지

1. 분류 체계 재정렬. Label / Category / Tag / Sticker 4-system을 → Type / Tag / Bundle 3-system으로 압축. 사용자 인지 부담 감소. 새 entity 추가 시에도 같은 패턴.

2. Detail Panel 강화. 단순 텍스트 메타데이터 → 통계 차트 + 미니 그래프 + Activity timeline. 한 노트의 "맥락"을 우측 패널에서 한눈에 파악.

3. Focus Mode 도입. Default / Focus / Zen / Compact 4단계. 작성에 몰입할 때, 정리할 때, 탐색할 때 각기 다른 UX. ⌘. 단축키로 즉시 전환.

진행 단계

Phase 1 (2-3주): 즉시 시각 개선 — 사이드바 Tags 인라인, Notes 테이블 압축, Detail Panel 차트, Wiki TOC. 데이터 모델 변경 0.

Phase 2 (3-5주): Focus Mode 시스템, Type 통합, Home 대시보드 위젯화.

Phase 3 (4-8주): Books 엔티티 신규, Timeline View, 3-Layer Architecture 완성.
