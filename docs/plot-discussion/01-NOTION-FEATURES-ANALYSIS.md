# 노션 기능 분석 — Plot에 가져올 것 / 버릴 것

> 2026-03-30 대화 기반 정리

## 배경

노션의 전체 기능 목록(나무위키 기준)을 검토하고, Plot의 철학(로컬퍼스트, 성능 우선, 구조적 단순성)에 맞는 것만 선별.

---

## 가져올 기능

### 에디터 레벨 (TipTap 확장으로 가볍게 구현 가능)

| 기능 | 설명 | 현재 Plot 상태 |
|------|------|----------------|
| Toggle block | 접기/펼치기. 긴 위키 문서에서 유용 | ✅ 구현됨 |
| Callout block | 아이콘 + 배경색 강조 박스 | ✅ 구현됨 |
| Simple table | 순수 마크다운 테이블 (DB 테이블 아님) | ✅ 구현됨 |
| Turn into | 블록 타입 변환 (h1↔h2, bullet↔numbered 등) | ❌ TODO P0 |
| Table of Contents | 헤딩 기반 자동 목차 | ✅ 구현됨 (플로팅 TOC + 인라인 TOC) |

### 내비게이션 레벨

| 기능 | 설명 | 현재 Plot 상태 |
|------|------|----------------|
| Breadcrumb | 폴더 깊이 + 현재 위치 표시 | ✅ 구현됨 |
| Favorites / Pin | 사이드바 상단 고정 | ✅ 구현됨 |

### 인라인 참조 레벨

| 기능 | 설명 | 현재 Plot 상태 |
|------|------|----------------|
| @멘션 (페이지/날짜) | 인라인 링크 + 캘린더 연결 | ✅ 구현됨 (노트/위키/태그/날짜 4종 통합) |

## 의도적으로 뺀 것들

| 기능 | 제외 이유 |
|------|----------|
| Synced Block | 동기화 상태 관리 복잡도 |
| 인라인 DB 뷰 (노션 방식) | 스키마 설계 강요 → 철학 충돌. 대안: 스키마 없는 쿼리 뷰 |
| 웹 임베드/북마크 (풍부한) | 외부 의존 + 렌더링 비용. 링크 카드로 대체 |
| 수식/롤업 | DB 전제 기능 |
| 실시간 협업 | 로컬퍼스트 + 싱글유저와 충돌 |
| AI 블록 | No-AI 원칙 |

## 노션 기능 전체 매핑 (나무위키 기준)

### 로컬에서 빠르게 구현 가능한 것들

| 구분 | 기능 | 로컬 구현 | 예상 속도 | Plot 상태 |
|------|------|----------|----------|-----------|
| 에디터 블록 | 텍스트/제목/목록/토글/콜아웃/인용/코드/수식/구분선/TOC | TipTap 확장 | 즉시 | ✅ 대부분 완료 |
| 페이지 구조 | 페이지 in 페이지, 링크, 멘션 | 폴더 + 크로스링크 | 즉시 | ✅ 완료 |
| 미디어 | 이미지/파일/오디오/비디오 | 로컬 저장 (IDB) | 즉시 | ✅ 완료 |
| DB 뷰 | 테이블/칸반/캘린더/갤러리 | 메타 기반 뷰 | 빠름 | ✅ List/Board/Calendar |
| 검색 | 전문 검색 | FlexSearch Worker | 노션보다 빠름 | ✅ 완료 |
| 템플릿 | 페이지 템플릿 | 노션보다 정교 | 즉시 | ✅ 완료 |
| 반출 | MD/HTML/CSV | 로컬이라 더 쉬움 | 즉시 | ❌ TODO P1 |

### 로컬에서 불필요하거나 대체 가능한 것들

| 구분 | 기능 | 로컬 대안 |
|------|------|----------|
| 임베드 | 외부 서비스 (Figma, CodePen 등) | 링크 카드 + 유튜브 iframe |
| 실시간 동기화 | 멀티유저 협업 | 싱글유저 → 불필요 |
| 범용 DB | 스키마 자유 정의 + 수식 + 롤업 | 메타데이터 기반 쿼리 뷰 |

## 결론

> 노션의 "모든 기능"이 아니라 "노션에서 사람들이 실제로 쓰는 기능"을 빠르게 만드는 것이 목표.
> 빠지는 건 범용 DB 스키마 자유 정의, 멀티유저 실시간 협업, 풍부한 임베드 — 이 세 개가 정확히 노션을 느리게 만드는 원흉들.
