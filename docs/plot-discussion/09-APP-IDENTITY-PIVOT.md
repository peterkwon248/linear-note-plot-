# Plot 정체성 재정의 — 노트앱에서 개인 워크스페이스로

> 2026-03-30 대화 기반 정리

---

## 핵심 전환

```
Before: "로컬퍼스트 노트앱으로서의 모든 것"
After:  "세상에서 가장 빠른 개인 워크스페이스. 쓸수록 발견이 많아지는."
```

---

## 왜 "노트앱"이 제약인가

기능은 이미 워크스페이스급 (63,000줄, 20 슬라이스). 하지만 새 페이지를 열면 항상 "노트 에디터"가 나온다. 그 순간 "이건 글 쓰는 앱이구나"로 인식됨.

노션은 새 페이지에서 "넌 여기서 뭐든 할 수 있어"를 말하고, Plot은 "여기에 글을 써"를 말하고 있었음.

---

## 바꿔야 할 것 — "새 페이지 경험"

아키텍처 변경이 아니라 **UX 레이어 변경**.

### 새 페이지 생성 첫 화면

```
+ 새 페이지 클릭 →

┌─────────────────────────────┐
│  빈 페이지로 시작             │
│                              │
│  ── 또는 템플릿 선택 ──       │
│                              │
│  📝 빈 노트                  │
│  📋 투두 리스트              │
│  📖 일기                     │
│  📊 칸반 보드                │
│  📚 위키 문서                │
│  📅 주간 계획                │
│  🔬 리서치 노트              │
│  💡 브레인스토밍             │
│  ...                         │
│                              │
│  최근 사용한 템플릿           │
│  [일기] [투두] [회의록]       │
└─────────────────────────────┘
```

이건 이미 있는 템플릿 시스템의 **노출 방식 변경**. 지금 `/template` 슬래시 커맨드에 묻혀있는 걸 새 페이지 생성의 첫 화면으로 올리는 것. 구현 비용: CreateDialog 컴포넌트 수정 수준.

### 템플릿이 "워크스페이스" 느낌을 만드는 원리

```
"투두 리스트" 선택 → Label:Todo 자동 세팅 → Todo 뷰에서 자동 수집
"일기" 선택       → Label:Journal 자동 세팅 → Calendar에서 날짜별 표시
"칸반 보드" 선택  → Label:Project 자동 세팅 → 인라인 쿼리 뷰 보드 모드 삽입
"리서치 노트" 선택 → Label:Research 자동 세팅 → Wiki에서 관련 문서 자동 연결
```

전부 같은 엔진(TipTap + Zustand + view-engine) 위에서 돌아감. 다른 건 Label + 초기 컨텐츠뿐. 속도 저하 요소 제로.

### 체감 자유도 전략

```
실제 자유도:  5개 레이아웃 × 10개 템플릿 × 3개 폰트 프리셋
체감 자유도:  "나만의 공간을 만들었다"

비유: IKEA 전략
노션 = 목공소 (진짜 자유, 근데 어려움)
Plot = IKEA (조합이 한정적이지만 충분히 예쁘고, 5분이면 완성)
```

## 네이밍: Notes → Pages

Activity Bar와 유저 대면 UI에서는 "Pages"로 표기. 내부 코드는 `Note` 타입 유지.

```
외부 명칭 (유저가 보는 것):  Pages
내부 명칭 (코드/데이터):     Note (63,000줄 리팩토링 불필요)
```

Activity Bar 아이콘 텍스트 + 사이드바 헤더만 변경.

## 페이지 레이아웃 프리셋

```
Standard    = 현재 Plot 에디터 (전폭 텍스트)
Wide        = 더 넓은 에디터 (사이드바 자동 접힘)
Side-by-side = 좌/우 분할 (기존 듀얼 패널)
Dashboard   = 인라인 쿼리 뷰 여러 개 배치
Journal     = 날짜 헤더 + 좁은 폭 + 세리프 폰트
```

구현: Note 메타에 `layout: "standard" | "wide" | "dashboard" | "journal"` 추가. 템플릿이 자동 세팅. CSS 변수 몇 개 + 에디터 wrapper의 max-width / padding 조정.

## 포지셔닝 — 경쟁 앱 대비

```
노션        = 팀 워크스페이스 (협업이 핵심, 속도는 희생)
옵시디언    = 파워유저 노트앱 (플러그인이 핵심, 진입장벽 높음)
Plot        = 개인 워크스페이스 (속도가 핵심, 온톨로지가 차별화)
```

유저별 소구점:
- 노션에서 오는 사람 → "노션인데 빠르고, 연결할 만한 걸 찾아준다"
- 옵시디언에서 오는 사람 → "플러그인 설치 안 해도 다 돼있다"
- Apple Notes에서 오는 사람 → "이렇게 심플한데 이렇게 강력해?"

---

## 온톨로지 철학 — 앱은 제안만, 결정은 유저

```
유저의 행동:     쓴다, 링크한다, 태그 단다 (의식적 행동)
앱의 역할:       "이거 연결해볼래?" 제안한다 (제안만)
유저의 선택:     수락 or 무시 (결정권은 항상 유저)
```

이미 코드에 이 원칙이 반영됨:
- `RelationSuggestion` — status: `pending | accepted | dismissed`
- `WikiClusterSuggestion` — 유저가 수락/거부
- `unlinked-mentions` — 보여주기만, 자동 적용 안 함
- `Discover 엔진` — 추천만, 자동 실행 안 함

**자동으로 하는 건 "발견"이고, 실행하는 건 항상 유저.**

앱이 유저 대신 판단하는 게 아니라, 유저가 더 좋은 판단을 할 수 있게 재료를 깔아주는 것. No-AI 원칙과 완전히 일치.

## 디자인 — 노션 수준으로 "느끼게" 하는 8단계

100% 노션 수준은 불가능 (노션 디자인팀 50명+). 하지만 "노션 수준으로 느껴지게"는 가능.

### 유저가 "노션 수준"이라고 느끼는 5가지

1. **타이포그래피가 예쁘다** → 폰트 + 사이즈 스케일 일관성
2. **여백이 넉넉하다** → 에디터 max-width + padding
3. **인터랙션이 부드럽다** → hover/transition 일관 적용
4. **아이콘/이모지가 풍부하다** → 페이지 아이콘 이모지 선택
5. **빈 상태가 안내한다** → 플레이스홀더 + 안내 문구

### 실행 계획 (기존 5-Phase + 추가 3-Phase)

```
Phase 1 → Toggle 통합              → 인터랙션 일관성
Phase 2 → Typography 표준화        → "예쁘다" 느낌
Phase 3 → Border/Grid 정리         → 정돈된 느낌
Phase 4 → Hover/Surface 통일       → 부드러운 느낌
Phase 5 → 아이콘/하드코딩 정리      → 완성도
Phase 6 → 에디터 여백/max-width     → "숨 쉬는" 느낌
Phase 7 → 페이지 아이콘 + 커버 이미지 → "내 것" 느낌
Phase 8 → 빈 상태 안내 문구         → "친절한" 느낌
```

전부 CSS + 메타데이터 필드 + 텍스트. 새 엔진 불필요.

## Activity Bar 최종

```
Home     = 대시보드 (뭘 할지 한눈에)
Pages    = 모든 페이지 (노트, 일기, 투두, 리서치, 뭐든)
Wiki     = 정리된 지식
Calendar = 시간축 (일정 + 투두 + 일기)
Graph    = 관계축 (모든 게 연결된 온톨로지)
```

Pages가 "노트"가 아니라 "내 모든 페이지". Todo는 별도 공간이 아니라 Pages 안에서 Label:Todo인 페이지들의 체크박스를 모아보는 사이드바 뷰.
