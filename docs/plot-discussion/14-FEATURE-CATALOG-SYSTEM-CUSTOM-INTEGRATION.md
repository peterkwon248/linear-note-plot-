# Plot 기능 카탈로그 — 시스템 & 커스터마이징 & 통합 & 접근성

> 2026-03-30 대화 기반 전체 정리.

---

## 입력 / 캡처

| 기능 | 설명 | 구현 비용 | 기존 인프라 |
|------|------|----------|-----------|
| Share Target (PWA) | 모바일에서 "공유" → Plot Inbox 직행 | 낮음 | PWA manifest |
| 음성 메모 | 마이크 버튼 → 녹음 → 오디오 블록 삽입 | 낮음 | MediaRecorder API |
| OCR (Tesseract.js) | 이미지 첨부 시 텍스트 추출 제안. 로컬 처리 | 중간 | Tesseract.js WASM |
| 이메일 → Inbox | 특정 주소로 이메일 전달 → 페이지 자동 생성 | 중간 | 로컬 API + Zapier |
| RSS 리더 → Inbox | RSS 피드 URL 등록 → 새 글 자동 Inbox 링크 카드 | 중간 | fetch + XML 파싱 |
| Readwise/Kindle 가져오기 | CSV/txt 드래그 → WikiQuote로 변환 → 출처별 정리 | 중간 | Import 확장 |
| CSV 드래그 앤 드롭 | CSV → TipTap Table 자동 변환 | 낮음 | Papa Parse |
| Google Calendar 읽기 전용 | iCal URL 구독 → Calendar에 외부 일정 표시 | 중간 | ical.js |

## 시스템 / 안전 / 복구

| 기능 | 설명 | 구현 비용 | 기존 인프라 |
|------|------|----------|-----------|
| 오프라인/PWA | 서비스 워커 + PWA manifest. 인터넷 없이 앱 실행 | 낮음 | Next.js PWA |
| Import/Export | Obsidian MD, Notion export, Evernote ENEX 가져오기 + MD/JSON 내보내기 | 중간 | 파서 |
| 로컬 백업/스냅샷 | 하루 한 번 자동 IDB 스냅샷 → 로컬 파일. "7일 전 복원" | 중간 | IDB export |
| 페이지 히스토리 (버전 관리) | 수정 시 이전 버전 스냅샷 보관. diff 뷰 | 중간 | IDB 스냅샷 |
| 충돌 복구 (Crash Recovery) | 비정상 종료 후 "저장되지 않은 변경사항 복구할까요?" | 낮음 | IDB 임시 저장 |
| 로컬 암호화 | AES 암호화. 앱 시작 시 비밀번호 입력 → 복호화 | 중간 | Web Crypto API |
| 앱 잠금 (App Lock) | 앱 시작 시 비밀번호/PIN. 전체 워크스페이스 접근 제어 | 낮음 | UI 레이어 |
| 인덱스 리빌드 버튼 | 검색/백링크/TaskIndex 전부 재생성. Settings에 배치 | 낮음 | 파생 데이터 재빌드 |
| 스토리지 사용량 대시보드 | "IDB: 45MB / 첨부파일: 230MB / 총: 275MB" 표시 | 낮음 | IDB 사이즈 조회 |
| PDF 예쁜 내보내기 | 현재 에디터 스타일 그대로 PDF. 커버+아이콘+폰트 반영 | 중간 | html2canvas + jsPDF |
| 페이지 공개 링크 | 읽기 전용 HTML로 내보내서 링크 공유 (스냅샷) | 중간 | Next.js API route |
| 기능 점진적 활성화 | Settings에서 Wiki/Todo/SRS/Autopilot 등 기능별 토글 | 중간 | lazy import |

| 커스텀 CSS 주입 | Settings > Appearance에 CSS 텍스트에리어 | 매우 낮음 | style 태그 삽입 |
| 에디터 폰트 사이즈 조절 | `Ctrl++` / `Ctrl+-` 실시간 조절 | 매우 낮음 | CSS 변수 |
| 밀도 설정 (Compact/Default/Comfortable) | 리스트 행 높이, 사이드바 간격 3단계 | 낮음 | CSS 변수 |

| 사이드바 섹션 순서 변경 | 드래그로 순서 변경 | 낮음 | dnd-kit |
| Activity Bar 아이콘 순서 변경 | 5개 공간 순서 유저 정의 | 낮음 | dnd-kit |
| 페이지 컬러 태그 | 리스트 왼쪽에 Label 색상 바 | 매우 낮음 | Label.color |
| 즐겨찾기 이모지 그루핑 | Pin 페이지들을 아이콘으로 시각적 그룹핑 | 낮음 | icon 필드 활용 |

## 우선순위 원칙

1. **디자인 폴리싱 8-Phase 먼저** — 기능 추가 전에 시각적 일관성 확보
2. **파생 데이터 > 새 엔티티** — 버그 최소화
3. **기능 토글** — 안 쓰는 기능은 결합 차단
4. **기존 인프라 재활용** — 새 엔진보다 기존 엔진의 표면 확장
5. **"라이트 유저가 헤비유저 착각"** — 시스템이 뒤에서 다 하고, 유저는 결과만
