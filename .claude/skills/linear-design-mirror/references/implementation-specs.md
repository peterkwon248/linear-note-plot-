# 구현 스펙 (Layer 3: HOW)

실제 CSS 값, 컴포넌트 패턴, 기술적 세부사항.
⚠️ 이 파일은 DevTools 캡처로 점진적으로 보강됨. 현재는 공식 소스에서 확인된 값만 포함.

## 확인된 구현 세부사항

### 타이포그래피
- 헤딩: **Inter Display** (표현력, 가독성)
- 본문: **Inter** (일반)
- 두 폰트를 구분하여 사용하는 이유: 헤딩에 더 많은 표현을 부여하면서 본문 가독성 유지

### 색상 시스템
- **LCH 색공간** 기반 (HSL이 아닌)
- 색상 생성: base 색상에서 operations를 적용하여 alias(surfaces, texts, icons, controls) 생성
- 2026 리프레시: chrome(blue) 사용을 제한하여 더 중립적 외관
- 대비 향상: 라이트모드에서 텍스트/아이콘을 더 어둡게, 다크모드에서 더 밝게

### UI 라이브러리
- **Radix UI** 기반
- Plot은 shadcn/ui 사용 (Radix UI 위에 구축되어 있으므로 호환)

### 앱 프레임워크
- **Electron** 기반 데스크톱 앱
- 네비게이션이 macOS, Windows, 브라우저 모두에서 동작해야 함
- PWA 지원 (모바일/태블릿)

### 싱크 엔진
- 프론트엔드-백엔드 실시간 싱크, 오프라인 지원
- 변경사항이 전송 불가 시 로컬 저장 후 연결 복구 시 재시도
- 사이드바에 "Syncing" + 대기 중 변경 수 표시

### 2026.03 UI 리프레시 세부 변경사항

changelog에서 추출한 구체적 변경:
1. **헤더/네비게이션/뷰 컨트롤**: 프로젝트, 이슈, 리뷰, 문서 간 일관성 확보
2. **아이콘**: 전체 리드로잉 + 리사이징
3. **사이드바**: 약간 더 어둡게 (dimmer)
4. **탭바**: 더 컴팩트, 둥근 모서리, 작은 아이콘/텍스트
5. **팀 아이콘**: 컬러 배경 제거
6. **경계선**: 더 부드럽고, 더 적고, 둥근 모서리

### 키보드 단축키 기술 세부사항
- 비US 키보드 레이아웃에서 `Cmd/Ctrl + <숫자>` 수정
- 문서 헤딩 단축키 표준화: macOS `Cmd Option 0-4`, Windows/Linux `Ctrl Alt 0-4`
- 코멘트 전송: `Enter` 또는 `Cmd/Ctrl Enter` 중 선택 가능 (Preferences)

## Plot 현재 토큰 참조

현재 DESIGN-TOKENS.md에 정의된 값들 (design-quality-gate 스킬에서 관리):

```
아이콘: strokeWidth 1.5, 표준 h-4 w-4, nav h-5 w-5, 작은 h-3.5 w-3.5
폰트: 15px(헤더/nav), 14px(body), 12px(보조), 11px(배지), 24px(에디터 제목)
트랜지션: duration-75, duration-150, duration-200 만 허용
스페이싱: px-4(헤더), px-2.5(사이드바), px-6(에디터), px-4(카드)
```

Linear DevTools 캡처 후 이 토큰들과 비교하여 차이점을 리포트할 것.
