# Surface: 컨텍스트 메뉴 (Context Menu)

## Layer 1: 디자인 의사결정

### 역할
4가지 액션 경로 중 하나. 우클릭으로 현재 요소에 가능한 모든 액션을 표면화.
"마우스를 선호하는 사용자를 위해 앱을 더 빠르고 쉽게 만든다."

### Safe Area (삼각형 안전 영역)
Andreas Eldh의 "Invisible details" 글 핵심.

서브메뉴가 있을 때의 문제: 마우스가 서브메뉴 방향으로 이동하는 중 다른 항목 위를 지나면 서브메뉴가 닫힘.

해결: 마우스 커서 위치와 서브메뉴 사이에 삼각형 safe area를 그린다.
- React 컴포넌트가 서브메뉴를 `ref`로 받아 크기/위치 측정
- 커서와 서브메뉴 사이에 SVG/CSS 삼각형을 동적으로 생성
- 삼각형 내부에서는 메뉴가 닫히지 않음
- Amazon, Khan Academy 등도 유사 패턴 사용, 네이티브 OS에서는 오래전부터 존재

### 가용 액션 (이슈 기준)
- 상태/우선순위 변경
- 담당자/라벨/예상치 설정
- 블로킹/블로커/중복 표시
- 사이클/프로젝트에 추가/제거
- Git 브랜치명 복사
- 이슈 URL/마크다운 복사
- AI 코딩 도구에서 열기 (Cursor, Claude Code 등)
- 삭제/아카이브

### 단축키 표시
각 메뉴 항목 오른쪽에 키보드 단축키 표시 → 커맨드 팔레트와 동일한 "학습 도구" 역할.

## Plot 매핑
- 노트/폴더에 대한 컨텍스트 메뉴 구현 시 safe area 패턴 필수
- shadcn/ui의 ContextMenu 컴포넌트 활용 (Radix UI 기반이므로 safe area 지원 가능)
- 각 항목에 단축키 표시
