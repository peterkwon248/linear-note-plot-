# Surface: 빈 상태 / 온보딩 (Empty States)

## Layer 1: 디자인 의사결정

### 빈 상태의 역할
단순히 "데이터가 없습니다"가 아니라, 사용자를 다음 행동으로 안내하는 기회.

### Linear의 접근
- Settings 페이지에 튜토리얼과 팁을 직접 배치 → 기능 발견의 장소
- 빈 뷰에서 해당 기능의 가치와 사용 방법을 간결하게 설명
- 명확한 CTA (Call to Action) 제공

### Linear 온보딩 가이드 구조
역할별 분기:
- Admin: 회사 규모에 따라 다른 가이드 (소규모 / 스타트업·중견 / 대규모)
- Team member: 기존 워크스페이스 합류 가이드

### Demo 워크스페이스
- 브라우저 로컬에서 동작, 새로고침 시 리셋
- 설정은 미포함, SLA 미지원
- 실제 이슈/프로젝트/워크플로우의 작동을 체험

## Layer 2: 구현 고려사항

### 빈 상태 체크리스트
모든 뷰에 대해:
- [ ] 데이터 없을 때의 표시 내용 정의
- [ ] CTA 버튼/링크 (행동 유도)
- [ ] 일러스트레이션/아이콘 (선택적, 과하지 않게)
- [ ] 짧고 명확한 설명 텍스트
- [ ] 기능의 가치 전달 (왜 이걸 쓰면 좋은지)

### DevTools 캡처 필요
- [ ] 빈 상태 컨테이너 레이아웃 (중앙 정렬 방식)
- [ ] 텍스트 크기/색상/정렬
- [ ] CTA 버튼 스타일
- [ ] 일러스트레이션/아이콘 크기

## Plot 매핑
- 노트 리스트가 비어있을 때: "첫 노트를 만들어보세요" + 단축키 안내
- 폴더가 비어있을 때: 드래그앤드롭 또는 생성 안내
- 검색 결과 없을 때: 검색어 수정 제안
- 온톨로지 뷰 첫 진입: 연결이 생기면 어떻게 보이는지 미리보기
