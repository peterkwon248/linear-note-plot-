# Surface: 커맨드 팔레트 (Command Palette, ⌘K)

## Layer 1: 디자인 의사결정

### 역할
"앱이 하는 모든 것을 손끝에 두는" 단일 진입점. 메뉴와 툴바가 차지할 화면 공간을 절약하면서도 모든 기능에 접근 가능하게 한다.

### Linear의 접근
Linear의 커맨드 팔레트는 **액션 중심**. 검색과 액션이 분리되어 있되, 같은 인터페이스에서 전환 가능.
- 기본: 커맨드/액션 모드
- 접두사(`i`, `p`, `u` 등)로 특정 엔티티 검색 모드 전환
- VS Code의 `>`(커맨드)와 일반(파일 검색) 전환과 유사한 패턴

### 핵심 디자인 결정
1. **컨텍스트 의존**: 현재 뷰에 맞는 커맨드가 먼저 표시. 사이클 뷰에서 열면 사이클 관련 커맨드 우선.
2. **학습 도구**: 각 커맨드 옆에 키보드 단축키 표시 → 자연스럽게 단축키 학습
3. **최근 사용**: 최근 사용한 커맨드가 상단에 → 반복 작업 가속

### James Robinson의 분석 (다크모드 관점)
- 모달 배경이 메인 배경보다 약간 밝음 → 전경(foreground) elevation 표현
- edge lighting으로 모달 경계를 정의 (다크모드에서 그림자 대신)
- 리스트 항목 하이라이트: 모달 대비 차이가 크지만 edge lighting 없음 → 같은 z-plane의 상태임을 전달

## Layer 2: 구현 스펙

### 동작 흐름
```
⌘K 입력
  → 모달 오픈 (오버레이 + 모달)
  → 입력 필드에 포커스
  → 최근 커맨드/이슈 표시
  → 타이핑 시 퍼지 매칭으로 실시간 필터
  → ↑/↓로 항목 이동
  → Enter로 실행
  → Esc로 닫기
```

### 접두사 필터 동작
```
[일반 타이핑] → 커맨드 + 엔티티 통합 검색
[i + space] → 이슈만 검색
[p + space] → 프로젝트만 검색
[u + space] → 사용자만 검색
```

### DevTools 캡처 필요 항목
- [ ] 모달 너비 (추정 ~640px)
- [ ] 모달 최대 높이
- [ ] 모달 border-radius
- [ ] 모달 배경색 (메인 대비 elevation 차이)
- [ ] 오버레이(backdrop) opacity
- [ ] 입력 필드 높이/패딩/폰트 크기
- [ ] 결과 항목 높이/패딩
- [ ] 키보드 단축키 배지 스타일 (bg, border-radius, font-size)
- [ ] 섹션 헤더 스타일 (Recent, Commands 등)
- [ ] 선택된 항목 하이라이트 색상
- [ ] edge lighting 값 (border 또는 box-shadow)

## Plot 매핑

Plot의 Search/Command Palette 서피스와 직접 대응.

현재 Plot 구현 상태:
- FlexSearch Web Worker 기반 전문 검색
- 기본 커맨드 팔레트 존재

Linear 패턴 적용 시 추가 고려:
- 접두사 필터 시스템 (`/note`, `/folder`, `/tag` 등)
- 컨텍스트 의존 커맨드 (현재 뷰에 따라 다른 커맨드)
- 각 커맨드 옆 단축키 표시
- 최근 사용 커맨드 추적
- elevation 기반 모달 배경색 (메인보다 약간 밝게)
