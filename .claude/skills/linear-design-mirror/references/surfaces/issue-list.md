# Surface: 이슈 리스트 (Issue List View)

## Layer 1: 디자인 의사결정

### 역할
Linear에서 가장 많이 보는 화면. 이슈를 스캔하고, 선택하고, 액션을 취하는 주요 작업 공간.

### 핵심 디자인 결정

**1. 정보 밀도 vs 가독성의 균형**
- 한 행에 많은 정보를 넣되, 시각적 노이즈를 최소화
- 핵심 정보(ID, 제목, 상태, 우선순위)는 항상 표시
- 부가 정보(라벨, 담당자, 예상치)는 Display Options로 토글

**2. 하이라이트 vs 선택의 이중 시스템**
- 하이라이트: 커서가 올라간 상태 → 단일 이슈 액션
- 선택(X): 체크박스 활성 → 다중 이슈 벌크 액션
- 이 구분 덕분에 키보드 네비게이션이 매끄러움

**3. 그룹핑의 유연성**
- 상태, 담당자, 프로젝트, 우선순위, 사이클, Focus 등으로 그룹핑
- Focus 그룹핑(My Issues): 긴급 → SLA → 블로킹 → 현재사이클 → 기타 순으로 자동 정렬
- 그룹 접기/펼치기: `T` 키 또는 토글 아이콘

**4. 리스트 ↔ 보드 전환**
- `⌘/Ctrl B`로 즉시 전환
- 리스트와 보드의 정렬은 연동됨 (독립적으로 설정 불가)
- 보드의 그룹핑은 상태가 기본이지만 변경 가능

### hover-reveal 패턴
마우스를 올려야 보이는 요소들:
- 체크박스 (왼쪽 끝)
- 인라인 액션 버튼들
- ... 메뉴
이를 통해 기본 상태에서의 시각적 노이즈를 줄임.

## Layer 2: 구현 스펙

### 리스트 행 구조
```
[체크박스(hover)] [상태아이콘] [ID] [제목...] [라벨] [담당자] [우선순위] [...메뉴(hover)]
```

### 키보드 인터랙션
| 키 | 동작 |
|----|------|
| `↑/↓` 또는 `J/K` | 하이라이트 이동 |
| `X` | 선택 토글 |
| `Shift ↑/↓` | 범위 선택 확장 |
| `⌘/Ctrl A` | 전체 선택 |
| `Enter` | 이슈 열기 |
| `Space` | Peek 미리보기 |
| `T` | 그룹 접기/펼치기 |
| `C` | 이슈 생성 |
| `F` | 필터 열기 |
| `⌘/Ctrl B` | 보드/리스트 토글 |

### Display Options 항목
- 레이아웃: 리스트/보드
- 그룹핑: Status, Assignee, Project, Priority, Cycle, Focus, Label, Label Group, SLA
- 서브그룹핑: 보드에서 swimlane
- 정렬: 수동, 우선순위, 생성일, 수정일, 상태
- 표시 속성: 선택적 토글

### 그룹 헤더
- 그룹명 옆에 이슈 수 또는 총 예상치 표시 (클릭으로 토글)
- `T`로 접기/펼치기

## Plot 매핑

Plot의 Note List 서피스와 대응.

적용 고려:
- J/K 네비게이션 + X 선택 패턴
- hover-reveal로 액션 버튼 숨기기 (기본 상태 깔끔하게)
- Display Options 패턴 (정렬, 그룹핑, 표시 속성)
- 그룹핑: 폴더, 라벨, 상태(NoteStatus), 생성일 등
- Peek(Space): 에디터를 열지 않고 노트 내용 미리보기
