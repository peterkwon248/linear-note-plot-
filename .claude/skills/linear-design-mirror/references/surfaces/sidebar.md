# Surface: 사이드바 (Sidebar)

## Layer 1: 디자인 의사결정

### 구조적 역할
사이드바는 Linear의 "inverted L-shape" 크롬의 수직 부분. 글로벌 네비게이션의 핵심.

### 2026.03 리프레시 핵심 변경
- 사이드바를 "몇 노치 더 어둡게" → 메인 콘텐츠가 시각적으로 우선
- 아이콘 크기 축소
- 팀 아이콘의 컬러 배경 제거
- 비활성 텍스트를 더 muted하게

이유: "사용자가 목적지에 도착하면 네비게이션은 배경으로 물러나야 한다."
사이드바가 너무 밝으면 메인 콘텐츠와 시각적으로 경쟁한다.

### 사이드바 구성 (위→아래)
1. **워크스페이스 스위처** — 좌상단, 드롭다운 메뉴
2. **인박스** — 알림 배지 포함
3. **My Issues** — Focus 그룹핑
4. **즐겨찾기 섹션** — 접기/펼치기, 폴더 지원
5. **팀 섹션** (각 팀별):
   - 팀명 (호버 시 ... 메뉴)
   - Issues (Active, Backlog)
   - Cycles
   - Projects
   - Views
   - Triage (활성화 시)
6. **하단**: Views, Drafts

### 키보드 인터랙션
- `G+_` 패턴으로 사이드바 항목 직접 이동
- 팀 간 이동: `O+T`로 팀 목록 오픈
- 즐겨찾기: `O+F`로 즐겨찾기 목록 오픈

### 접기 동작
- 접히면 아이콘만 표시
- 호버하면 임시로 펼쳐짐
- 2026 changelog: "접힌 사이드바가 호버 시 열리지 않는 버그" 수정됨

## Plot 사이드바와의 매핑

Plot 사이드바 현재 구조:
1. 검색/커맨드 팔레트 트리거
2. 빠른 액션 (새 노트 등)
3. 폴더 트리
4. 라벨/태그 섹션
5. 설정

Linear 패턴 적용 시 변경 고려:
- 인박스(SRS 리마인더) 위치를 최상단으로
- 즐겨찾기 섹션 추가 (자주 접근하는 노트/폴더)
- 사이드바 접기 시 아이콘만 표시 + 호버로 임시 펼치기
- "Syncing" 상태 표시 (오프라인→온라인 전환 시)
