# Surface: 인박스 / 트리아지 (Inbox & Triage)

## Layer 1: 디자인 의사결정

### 인박스 — 개인 알림 허브
- 사이드바 최상단 위치 (가장 자주 접근하는 뷰)
- `G → I`로 어디서든 즉시 이동
- **스플릿 뷰**: 왼쪽 알림 리스트 + 오른쪽 이슈 상세
- 알림을 클릭하면 "특수 인박스 뷰"에서 이슈를 보여줌 — 인박스 액션(삭제, 스누즈)과 이슈 수정을 동시에 가능

### 트리아지 — 팀 인박스
- 팀별로 존재, 사이드바에서 팀명 아래 위치
- 외부에서 들어오는 이슈(인테그레이션, 다른 팀)를 수락 전 검토
- 트리아지에 있는 이슈는 기본적으로 일반 뷰에서 제외 (워크플로우 외부로 간주)

### 스누즈 패턴
알림을 일시적으로 숨김. 지정 시간 또는 새 활동 시 재등장.
트리아지에서도 동일 패턴 — "지금은 아닌데 잊고 싶지 않은" 이슈.

### Triage Intelligence (AI)
- 전용 모듈에서 표시 — 유용하되 노이즈는 추가하지 않게
- 제안(담당자, 라벨 등)이 Linear의 기존 시각 언어를 사용 → 자연스럽게 블렌딩
- 로딩 상태: 반응성 있게 느끼되 추론 과정을 보여줘 신뢰 구축

### 키보드 인터랙션
| 키 | 동작 |
|----|------|
| `G → I` | 인박스로 이동 |
| `J/K` | 알림 간 이동 |
| `Backspace` | 알림 삭제 |
| `Shift Backspace` | 전체 삭제 |
| `U` | 읽음/안읽음 토글 |
| `Option/Alt U` | 전체 읽음/안읽음 |
| `H` | 스누즈 |
| `Shift S` | 구독 해제 |
| `⌘/Ctrl F` | 인박스 내 검색 |
| `⌘/Ctrl D` | 읽은 알림 전체 삭제 |

## Plot 매핑
- SRS 리마인더 시스템이 인박스 패턴과 유사
- 스누즈 = SRS에서 "나중에 다시" 개념
- Autopilot Engine의 자동 제안이 Triage Intelligence와 유사한 위치
- 스플릿 뷰 패턴: 왼쪽 리마인더 리스트 + 오른쪽 노트 미리보기
