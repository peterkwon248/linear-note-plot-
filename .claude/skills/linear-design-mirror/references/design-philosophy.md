## 크래프트 사이클

Karri Saarinen의 Config 2025 키노트 핵심:

크래프트는 순환한다: 수공예 → 기술 등장 → 대량 생산(크래프트 잊음) → 갈망 → 크래프트 부활.
AI 시대에 이 순환이 다시 일어나고 있다. AI는 노동만이 아니라 판단과 취향까지 아웃소싱하려 한다.
하지만 크래프트는 본질적으로 아웃소싱할 수 없다 — "the deliberate attention put into making something excellent, not because someone is checking, but because it matters to the maker."

**Quality = Belief + Care + Craft**
- Belief: 품질이 근본적으로 중요하다는 신념
- Care: 사용자 경험에 깊이 관심을 가지는 자세
- Craft: 그것을 인식할 수 있는 기술과 취향

Christopher Alexander의 "이름 없는 품질(quality without a name)" — 정확히 설명할 수 없지만 있을 때와 없을 때를 안다. 문이 완벽하게 열리고 닫히는 것 같은 만족감.

→ **Plot 적용**: 모든 인터랙션에서 "이게 맞는 느낌인가?"를 묻기. 메트릭보다 직감을 우선.

## 리디자인 철학

### 2024 리디자인 (Part I: Why)

- 디자인 부채는 매일 조금씩 쌓이지만, 한 번에 크게 갚아야 한다
- 엔지니어링과 다르게, 디자인은 홀리스틱 — 하나의 모듈만 업데이트하면 전체가 더 불일치
- 빠르게 진화하는 제품은 2-3년마다 디자인 부채를 갚아야 한다
- CEO의 참여를 끌어내려면 리디자인을 회사의 방향 전환과 연결해라
  (Airbnb 모바일 리디자인 → 회사의 mobile-first 전환 계기)

### 2024 리디자인 (Part II: How)

- inverted L-shape (사이드바 + 상단 크롬)에 집중
- 매일 하나의 완전한 화면 세트를 디자인
- "이 컨셉트 카가 얼마나 현실적일 수 있을까?"를 계속 물음
- 두 명이 동시에 다른 디자인 파트를 작업 → 대화와 결정 가속
- 오후마다 엔지니어 2인조로 코딩 → 디자이너는 다음 파이프라인 준비
- feature flag로 내부 테스트 → private beta → GA
- 리디자인은 빨리 해야 한다. 안 그러면 모든 프로젝트를 블로킹하고 새 기능이 바로 리디자인 대상이 된다

### 2026.03 리프레시

- "calmer interface" — 차분한 인터페이스
- 원칙 1: 주의를 벌지 않은 것은 주의를 요구하지 마라
  → 사이드바를 몇 노치 더 어둡게, 탭바를 더 컴팩트하게
- 원칙 2: 구조는 느껴져야지 보여서는 안 된다
  → border를 부드럽게, separator를 줄이기, 모서리를 둥글게
- 아이콘을 리드로잉, 크기 축소, 불필요한 시각 처리(컬러 팀 아이콘 배경) 제거
- Claude Code로 dev toolbar 안에 색상 도구를 만들어 실시간 토큰 실험
- 팔레트 확정 후 JSON으로 Figma 플러그인에 직접 임포트

→ **Plot 적용**: 리디자인 시 "한 번에 크게" 접근. feature flag로 비교 가능하게.

## 정보 계층 원칙

### opacity 계층 체계
Karri는 탐색 단계에서 흑백의 opacity 값으로 작업. 이를 통해 요소 간 elevation과 hierarchy를 빠르게 잡음.

핵심: 색상이 아닌 불투명도로 정보의 중요도를 표현.
- 가장 중요한 텍스트: 높은 opacity (0.93)
- 보조 텍스트: 중간 opacity (0.65)
- 힌트/카운트: 낮은 opacity (0.35)
- 비활성 아이콘: 더 낮은 opacity (0.45)

### 배경 elevation
다크모드에서 z-axis가 높을수록 더 밝다 (물리적 빛 메타포).
- 메인 배경 < 사이드바 < 모달 < 팝오버
- James Robinson: "Linear의 ⌘K 모달은 배경보다 약간 밝게 하여 전경감을 줌. 리스트 항목의 하이라이트는 모달 배경 대비 차이가 훨씬 크지만, edge lighting이 없어 같은 z-plane임을 전달"

### spacing으로 구분
border/divider 대신 gap으로 영역을 구분.
- 섹션 간: 미묘한 gap (8px 추정), divider 없음
- 밀도(density): 항목 간 간격 최소화, 정보 밀도 높임
- "덜어내기": border를 제거하면 더 깔끔해진다

→ **Plot 적용**: border 사용 전에 항상 "spacing만으로 충분한가?" 물어보기.

## 설정 디자인 철학

Adrien Griveau (Linear 디자이너)의 글 핵심:

"설정은 디자인 실패가 아니다" — 제품 설정과 사용자 선호는 다르다.
- 제품 설정: 제품이 올바르게 기본값을 잡아야 함 (이건 디자인 실패일 수 있음)
- 사용자 선호: 취향과 편안함의 문제 (이건 정당한 커스터마이징)

설정 페이지를 교육 도구로 활용: 튜토리얼과 팁을 설정에 직접 배치.
작은 디테일이 사용자를 떠나게 만든다 — "if your users don't feel comfortable using your product, even if it's due to small details, they quickly leave you for a more cozy place."

→ **Plot 적용**: 설정을 제품 발견의 장소로 만들기. 커스텀 테마, 단축키 설정은 정당한 기능.

## 품질 문화

### Quality Wednesdays

Tuomas Artman이 시작. 매주 수요일 각 엔지니어가 "버그는 아니지만 전체 경험의 품질을 떨어뜨리는" 작은 결함을 찾아 수정.

핵심 인사이트:
- 품질 추구는 개인 스포츠가 아니다 — 자기가 만든 UI의 미묘한 문제는 자기가 못 본다
- 그룹으로 하면 각자 다른 것을 발견한다
- 품질은 습관이어야 한다 — 양치질처럼, "시간이 없다"고 하지 않는 것

예시 수정들:
- 인접 버튼의 크기 표준화
- 이슈 작성기에서 줄 추가 시 높이 변화 방지

→ **Plot 적용**: 코드 리뷰 시 "이 인터랙션이 정말 right한가?" 체크 습관화.

### Zero Bugs Policy
이슈는 7일 이내 수정. MVP는 내부 전용 — 반쯤 만든 경험은 절대 출시하지 않음.

## AI 시대의 디자인

Karri의 "Design for the AI age" 핵심:

전통적 UI는 결정론적(deterministic) — 버튼 클릭 → 예측 가능한 결과.
AI는 비결정론적 — 같은 프롬프트에 다른 결과.

"디자인의 역할은 기능에 형태를 부여하는 것(form around function)."
AI 없는 form은 방향 없는 강.
Chat 인터페이스는 매우 약한(weak) form — 아티팩트로 보완하는 건 반창고.

더 나은 접근: 전통적 UI로 기능적 앱을 만들고, AI 기능으로 보완.
소프트웨어는 "workbench" 메타포 — 도구를 올려놓는 환경. AI는 그 위의 강력한 새 도구.

→ **Plot 적용**: 에디터가 workbench, AI 서제스천은 그 위의 도구. Chat이 아닌 컨텍스트 패널에서 표면화.

## Karri Saarinen의 10가지 규칙

Figma Blog 인터뷰 기반:

1. **품질 우선 문화는 위에서부터** — CEO가 톤을 설정해야
2. **작은 고품질 팀** — 3-5명이 대규모 위원회보다 나은 결정
3. **크래프트 지향 인재 채용** — 이미 좋은 결과물을 만드는 사람
4. **연결된 팀** — PM→디자이너→엔지니어 사일로 금지, 모두가 품질 책임
5. **데이터가 아닌 직관** — A/B 테스트 대신 직관을 신뢰하되, 실수에서 배움
6. **핸드오프 없음** — "dev에 넘기기" 없이 끝까지 함께
7. **고정 팀 없이 순환** — 같은 영역만 계속 하면 아이디어가 고갈
8. **사용자를 놀라게 하라** — 데이터나 사용자 자신조차 말해줄 수 없는 것을 만들어
9. **놀라움을 위해 실험** — 틀릴 수 있지만 그 실수가 사고 실험이 됨
10. **사용자 org chart은 신경 안 쓴다** — 경험만 신경 쓴다
